# AI模型集成指南

## 概述

小白云简历生成助手现在支持集成真实的AI模型！本指南将帮助您配置和使用AI功能。

## 功能特点

✅ **本地资料优先**：先检索用户导入的资料
✅ **AI智能回答**：结合本地资料和AI模型生成专业回答
✅ **优雅降级**：AI不可用时自动使用本地响应
✅ **简历专业**：专注于简历生成相关问题

## 快速开始

### 1. 安装后端依赖

```bash
cd backend
npm install
```

### 2. 配置API密钥

复制 `.env.example` 为 `.env`：

```bash
cp .env.example .env
```

然后编辑 `.env` 文件，填入您的OpenAI API密钥：

```env
# OpenAI API配置
OPENAI_API_KEY=sk-your-actual-api-key-here
OPENAI_API_BASE=https://api.openai.com/v1
OPENAI_MODEL=gpt-3.5-turbo

# 服务器端口
PORT=3001
```

### 3. 启动后端服务

在 `backend/` 目录下运行：

```bash
npm start
```

或使用开发模式（支持热重载）：

```bash
npm run dev
```

### 4. 启动前端

在项目根目录下运行：

```bash
npm run dev
```

## 使用说明

### 导入资料

1. 在左侧侧边栏点击"导入资料"
2. 选择导入类型（个人资料、教育经历、工作经验、项目经历、专业技能）
3. 填写相关信息并保存

### 与AI对话

1. 打开聊天界面
2. 输入您的问题或需求
3. AI会自动：
   - 检索您导入的本地资料
   - 结合专业知识回答
   - 提供简历相关建议

**场景1：扩写简历**
> 用户：请帮我扩写简历
> 
> AI：根据您导入的资料，我将为您生成专业的简历内容...

**场景2：生成简历**
> 用户：帮我生成一份申请前端开发工程师的简历
> 
> AI：好的！根据您的资料，我为您创建了一份专业的简历...

**场景3：简历建议**
> 用户：我的简历应该怎么优化？
> 
> AI：根据您的资料，我有以下建议...

## 配置说明

### API配置选项

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| OPENAI_API_KEY | OpenAI API密钥 | 必填 |
| OPENAI_API_BASE | API基础地址 | https://api.openai.com/v1 |
| OPENAI_MODEL | 使用的模型 | gpt-3.5-turbo |
| PORT | 后端服务端口 | 3001 |

### 支持的模型

- gpt-3.5-turbo（推荐，性价比高）
- gpt-4（更强大，成本更高）
- 其他兼容OpenAI API的模型

## 架构说明

```
前端 (Vue 3)
    ↓
后端代理 (Express)
    ↓
AI API (OpenAI/其他)
    ↓
返回智能回答
```

### 本地资料检索流程

1. 用户发送消息
2. 系统检索用户导入的所有资料：
   - 个人信息
   - 教育经历
   - 工作经验
   - 项目经历
   - 专业技能
3. 将本地资料和用户消息一起发送给AI
4. AI结合资料生成专业回答
5. 返回给用户显示

## 故障排除

### 问题1：无法连接到后端服务

**错误：`无法连接到AI服务，使用本地响应`

**解决方案：
1. 确认后端服务已启动：`cd backend && npm start`
2. 检查端口3001是否被占用
3. 检查防火墙设置

### 问题2：API调用失败

**错误：`API请求失败`**

**解决方案：**
1. 检查 `.env` 文件中的API密钥是否正确
2. 确认API密钥有足够的配额
3. 检查网络连接
4. 尝试更换API基础地址

### 问题3：AI没有使用本地资料

**解决方案：**
1. 确认已导入并保存了资料
2. 检查资料是否完整填写
3. 刷新页面重新加载

## 不使用API密钥

如果您没有API密钥，系统仍可正常使用：

1. 系统会自动使用内置的本地响应逻辑
2. 提供基础的简历生成功能
3. 支持本地资料扩写
4. 无需额外配置

## 安全建议

1. **不要提交 `.env` 文件到版本控制
2. 使用环境变量管理API密钥
3. 定期轮换API密钥
4. 监控API使用量和费用
5. 在生产环境中使用反向代理
